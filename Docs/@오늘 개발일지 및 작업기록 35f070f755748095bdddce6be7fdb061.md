# @오늘 개발일지 및 작업기록

날짜: 2026년 5월 14일
단계: Phase 5

📝 Phase 5. 매치 4 퍼즐 시스템 체크리스트

1. 탐색 트리거(Trigger) 구현

블록이 낙하를 멈추고 `Idle` 상태가 되는 순간(`FVector::Dist` 판정 직후) 체크 함수 호출 로직 작성

블록 파괴 후 위쪽 블록들이 떨어져서 정착했을 때도 연쇄적으로 체크가 작동하는지 확인

2. 데이터 구조 및 헬퍼 세팅

상/하/좌/우 4방향 탐색을 위한 `TArray<FVector> CheckDirs` 선언 및 초기화

중복 탐색 방지를 위해 방문한 블록을 저장할 `TSet<ABlock*>` 변수 활용 로직 설계 (성능 및 중복 방지 최우선)

블록의 '색상' 또는 '클래스'를 비교하기 위한 기준 변수 확립

3. 플러드 필(Flood Fill) 핵심 알고리즘 구현

**재귀(Recursive)** 또는 **큐(Queue)** 방식을 선택하여 인접한 동일 색상 블록 탐색 함수 작성

탐색 시 `GetActorInDirection` 헬퍼 함수를 활용하여 상하좌우 액터 포인터 획득

탐색된 블록들을 임시 배열(MatchedBlocks)에 수집

4. 파괴 로직 및 연쇄 반응

수집된 블록의 개수가 4개 이상인지 판별 (`MatchedBlocks.Num() >= 4`)

조건 만족 시, 수집된 모든 블록의 `OnInteracted()` (또는 전용 파괴 함수)를 일괄 호출

블록 파괴 후 발생하는 중력 이동이 다시 새로운 매치를 만들어내는지(연쇄 반응) 루프 구조 점검

5. 예외 처리 및 디버깅

이미 파괴 중인 블록이 탐색 대상에 포함되지 않도록 방어 코드 작성

에디터 상에서 레이캐스트(Debug Line)를 통해 탐색 범위가 정확한지 시각적 확인

### 💡 내일 작업을 위한 멘토의 팁

내일 가장 중요한 포인트는 `TSet`입니다. 오늘 리팩토링하시면서 배열과 반복문을 줄이셨던 것처럼, 퍼즐 탐색에서도 "이미 확인한 블록인가?"를 체크할 때 `TArray::Contains`보다 `TSet`을 사용하는 것이 검색 속도 면에서 훨씬 유리합니다.

또한, 연쇄 반응(Combo)이 일어날 때 블록들이 한꺼번에 터지는 연출과 순차적으로 터지는 연출 중 어떤 것이 기획 의도에 맞을지도 내일 코딩하면서 함께 고민해 보시면 좋을 것 같습니다.
