- **Issue 3. 덜덜 흔들리는 블록 틈새 끼임 버그 (타이밍 이슈)**
    - **원인:** 블록이 낙하 전 `Anticipating`(대기) 상태일 때 캐릭터가 그 밑으로 들어가면, 아직 `Falling` 상태가 아니어서 Overlap 이벤트의 사망 판정이 무시됨. 이후 낙하가 시작되어도 이미 겹쳐진 상태라 Overlap 이벤트가 갱신되지 않아 캐릭터가 죽지 않고 블록에 끼어버림.
    - **해결:** 블록이 대기를 끝내고 `StartFalling()`을 선언하는 1프레임 찰나의 순간에, `IsOverlappingActor()`를 호출. 배열 순회나 불필요한 메모리 할당 없이 오직 0번 플레이어 폰(Pawn)과 직접 겹쳐있는지만 초경량으로 검사하여 끼여있는 플레이어를 처형하도록 방어 코드 추가.
- 교통체증 버그, 블랙홀 버그
    - 블록 매치 및 연쇄 파괴를 구현하면서 기존에 레이캐스트를 사용해서 tick으로 관리하는건 미세하게 차이가 날경우에는 떨어지다가 공중에서 멈추는 버그가 확인됨
    - 코드를 수정해서 조건을 늘리면 블랙홀(제일 하단에 블록이 모이는 현상)이 나옴
    - 해결방법은 그리드 기반 FloodFill 방식으로 코드를 리팩토링 하는 거였음
