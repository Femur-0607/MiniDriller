# @어제 개발일지 및 작업기록

날짜: 2026년 5월 13일
단계: Phase 4

**Phase 4. 중력 및 블록 낙하 시스템 (Gravity & Block Falling)**

**1. 블록 하단 감지 및 등속 낙하 시스템 구현**

- 레이 캐스트를 사용해서 하단 블록 감지 및 낙하 시스템
- Tick 내부에서 FMath::VInterpConstantTo를 활용한 부드러운 낙하(Lerp) 적용
- 파괴 시(OnInteracted) 바로 위 블록을 레이캐스트로 감지하여 연쇄 낙하 신호(CheckAndFall) 전달 로직 추가
- ABlock에 EBlockState (Idle, Anticipating, Falling) 상태 머신 도입

**2. 부드러운 낙하 이동 (Lerp / Interp)**

- 등속 보간(MoveTowards)으로 다음 프레임 위치 계산 (이동 속도는 임의로 300.f 설정)
`FVector nextLocation = FMath::VInterpConstantTo(currentLocation, targetLocation, DeltaTime, 300.f);`

**3. 연쇄 낙하 트리거 (Chain Reaction)**

- 파괴될경우 상태를 변화해서 떨어지는상태에서는 레이캐스트를 위로 쏴서 연쇄적으로 블록 낙하 트리거를 구현
- 레이캐스트로 구현한 이유는 이후 추가기능인 4개 이상이면 파괴하는 원본 게임의 기능을 구현할때 레이캐스트로 구현하는게 더 용의하기 때문

**4. 플레이어 압사 판정 (Player Crush)**

- 블록이 낙하 전 **대기(Anticipating) 상태일 때** 플레이어가 미리 밑으로 들어가면 `NotifyActorBeginOverlap`이 1회만 호출되어 압사 판정이 무시되는 끼임 버그가 발생함.
    - 이를 해결하기 위해 낙하가 진짜 시작되는 **`StartFalling()` 첫 프레임에 `IsOverlappingActor` (또는 GetOverlappingActors)를 통해 겹친 플레이어를 한 번 더 검사**하도록 수정함.
1. 아키텍처 개선: 전역 상태 관리자 (Subsystem) 도입
- 플레이어 사망 시 캐릭터가 직접 게임 오버를 처리하지 않고, `UGameStatusSubsystem`(`UWorldSubsystem` 상속)을 생성하여 **"나 죽었어!"라고 이벤트(Delegate)만 방송(Broadcast)** 하도록 구현.
- 객체 간 결합도를 낮추고 데이터 주도(Data-Driven) 설계를 적용함.
