# @내일 개발일지 및 작업기록

날짜: 2026년 5월 15일
단계: Phase 6

### **1. 전역 상태 매니저 (GameStatusSubsystem) 구축**

- **클래스 생성:** `UGameInstanceSubsystem`을 상속받는 `UGameStatusSubsystem` C++ 클래스 생성하기. (레벨이 바뀌어도 데이터가 유지되도록 Subsystem 활용)
- **핵심 변수 선언:** `CurrentOxygen`(float), `CurrentDepth`(int32), `CurrentLevel`(int32), `TotalScore`(int32), `bIsPlayerDead`(bool) 선언.
- **타이머 로직 구현:** 게임 시작 시 타이머를 작동시켜 1초마다 `CurrentOxygen`을 감소시키는 `ConsumeOxygen()` 함수 실행.
- **게임 오버 처리:** 산소가 0 이하로 떨어지면 `bIsPlayerDead`를 true로 바꾸고 `OnPlayerDied` 델리게이트(이벤트) 방송하기.

### **2. 특수 블록 다형성(Polymorphism) 구현**

- **클래스 상속:** `ABlock`을 부모로 두는 자식 클래스 `AObstacleBlock`(장애물), `AItemBlock`(아이템) 생성.
- **가상 함수 오버라이드:** 자식 클래스에서 `OnInteracted()` 함수를 덮어쓰기(Override).
    - **장애물:** 타격 시 Subsystem의 산소를 강제로 -10% 깎는 페널티 적용.
    - **아이템:** 타격 시 Subsystem의 산소 회복 및 `AddScore()` 호출.
- **통신 연동:** 블록들이 방금 만든 `UGameStatusSubsystem`을 캐싱하여 직접 데이터를 조작할 수 있도록 연결.

### **3. 깊이(Depth) 시스템과 레벨 스케일링**

- **깊이 증가 연동:** 맵 매니저가 라인을 재배치(`RecycleTopLine`)할 때마다 Subsystem의 `CurrentDepth` 증가시키기.
- **레벨업 감지:** `CurrentDepth`가 100의 배수를 돌파할 때 `CheckLevelUp()` 함수를 통해 `CurrentLevel` 상승.
- **난이도 조절:** 레벨이 오를수록 맵 매니저가 블록을 재배치할 때 '장애물 블록'이 뜰 확률(FMath::RandRange)을 높이고, 초당 산소 소모량을 가속하기.

### **4. 인게임 데이터 바인딩 (HUD 시각화)**

- **WBP_InGameHUD 생성:** 게임 플레이 중 항상 떠 있을 UI 화면 만들기.
- **UI 요소 배치:** 산소량을 나타낼 프로그레스 바(Progress Bar)와 깊이/점수를 나타낼 텍스트(Text) 배치.
- **데이터 연결:** UI의 Tick(또는 이벤트 기반 델리게이트)을 통해 Subsystem의 `CurrentOxygen`과 `TotalScore` 값을 가져와 화면에 실시간으로 그리기.
